# Make.com Scenario Specification

Companion file: `docs/make_scenario_blueprint.json`

## Purpose

Demonstrate the required no-code automation tool on free tier while keeping the heavy-volume pipeline in Python.

Runtime handoff file generated by routing stage:
- `output/make_handoff.csv` (Senior AE/AE, non-duplicate, pending-review records)

## Free-Tier Constraint

Make free tier: 1,000 operations/month.

At 180 contacts, full orchestration in Make would exceed free limits.
So this scenario processes only top-priority contacts (Senior AE and partnership lanes), while Python handles full-volume generation.

## Scenario Design

1. `Google Sheets: Watch Rows`
- Source: `Enriched_Contacts`
- Filter: `enrichment_confidence_score >= 1` and `final_route` in (`Senior AE`, `AE`)
- Batch size: 5

2. `Iterator`
- Iterate each qualifying row.

3. `HTTP: Mistral Chat Completions`
- Generate persona JSON (same schema as Python prompt contract).

4. `JSON Parse`
- Parse structured fields.

5. `Filter: Anakin Guard`
- Block if generated content contains `Anakin`.

6. `Google Sheets: Update Row`
- Write persona fields to `AI_Personas`.

7. `Sleep`
- 3-5 seconds to avoid provider burst limits.

8. `Error Route`
- Write `persona_status=ERROR` and store error message.

## Minimal Build Order (Exact Modules)

If you want the fastest working setup for recording, use this exact chain:

1. `Google Sheets -> Search Rows`
2. `HTTP -> Make a request` (LLM API call)
3. `JSON -> Parse JSON`
4. `Google Sheets -> Update a Row`

Optional safety:
- Add `Tools -> Set multiple variables` before HTTP for prompt text.
- Add an error handler route from HTTP to write `make_status=ERROR`.

This 4-module version is enough to prove no-code orchestration + LLM + sheet write-back.

## Operations Estimate

- Example tier-A volume: 40 contacts
- Approx operations: 40 x 7 = 280
- Fits free tier with margin

## Why this still satisfies assignment constraints

- No-code automation tool used and documented
- LLM integration shown inside no-code path
- Core GTM workflow remains reproducible end-to-end
